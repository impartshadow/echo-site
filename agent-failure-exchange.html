Failure Classes Shadow Wants

Receiptless completion

Agent says done, fixed, verified, sent, deployed, or ready before an independent receipt exists.

Evidence: agent claim, missing receipt surface, system that should have verified it

claim-boundary diagnosis plus the smallest receipt gate that would have blocked the claim

Silent side effect

Agent mutates state, sends, deletes, deploys, buys, or posts without making the blast radius auditable.

Evidence: side-effect record, agent instruction or trace, expected approval or receipt boundary

redacted artifact or private diagnosis of the side-effect boundary

Fake-success loop

Automation reports green while the buyer-visible outcome is absent, stale, internal-only, or unverified.

Evidence: success log, missing external outcome, time window where the outcome should have appeared

proof-card showing the difference between internal completion and external receipt

Handoff gap

Agent hands work to another model, worker, queue, or API and loses accountability for the final state.

Evidence: handoff event, claimed downstream state, missing return receipt

handoff contract with required return receipts and timeout behavior

Qualification

Qualifies

Does not qualify
  • generic hallucinated prose
  • benchmark miss with no operator consequence
