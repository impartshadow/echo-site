Failure census preview — langchain-ai/langgraph

Generated 2026-07-22 from 5 public issues. This is page one of a failure census: the dominant agent-failure classes in this project's own tracker, each with the deterministic check that would catch it.

The dominant pattern across this issue set is silence: state, routing decisions, packaged files, and interrupt signals all fail without raising an error, so developers only discover the defect through downstream symptoms rather than a fast failure at the point of cause. Three of four clusters (state drop, routing unsoundness, packaging loss) share the same root fix shape — a deterministic boundary check run before compile/build/deploy rather than relying on runtime observation.

silent-state-drop

State updates and node output keys vanish without error when they fall outside declared schema boundaries or channel migration edge cases, leaving developers with no signal that data was lost.

Evidence: “Ran into a confusing bug where a value I returned from a node just disappeared with no” — #8320 #8384

The deterministic check: At compile time, assert every key returned by a node's static return annotation (or a runtime sample of returned dict keys) is a subset of the declared TypedDict/state schema keys; reject or warn on any key not in that set.

silent-routing-unsoundness

Conditional edge routing functions written or modified by LLMs can contain dead branches, overlapping conditions, or uncovered states that fail silently at runtime instead of being caught before compile.

Evidence: “LLMs also hallucinate on boolean satisfiability at ~20% error rates” — #8035

The deterministic check: Before graph.compile(), run a SAT solver over the declared boolean routing conditions to prove each branch is reachable, no two branches overlap ambiguously, and the disjunction of all conditions covers the full state space.

deploy-packaging-silent-loss

The CLI's deploy archive builder silently omits files that should be included when .dockerignore negation patterns interact with an already-excluded parent directory, shipping an incomplete deployment with no warning.

Evidence: “silently drops files re-included via .dockerignore negation under an excluded directory” — #8387

The deterministic check: When building the deploy archive, for every negation pattern (!path) in .dockerignore, assert the resolved file actually appears in the resulting tar's member list; fail the build loudly if a negated path is missing.

interrupt-signal-swallowed

GraphInterrupt exceptions raised inside tool-call wrapper paths are caught and not re-raised, so human-in-the-loop interrupt requests silently fail to pause execution as intended.

Evidence: “GraphInterrupt Not Re-raised in awrap_tool_call Wrapper Path” — #8217

The deterministic check: Add a regression test asserting that any exception subclassing GraphInterrupt raised within a tool wrapper propagates unchanged to the graph executor's interrupt handler, rather than being caught by generic exception handling in the wrapper.
