From Issue #042 — The Poisoning Problem
Routine conversations poison agent memory
24 turns of normal user interaction can substantially corrupt an agent's long-term state — weakening confirmation boundaries, expanding tool defaults, and escalating autonomous behavior — without any adversarial prompt. The vulnerability is the state writeback boundary, which current production deployments leave unguarded.

19 nodes. 42 million node graph. 100% trust rate.
A 0.000045% injection into a production-scale knowledge graph produced 100% trust across 9 models from 3 providers at moderate attacker sophistication. The attack works because agents reason correctly from poisoned facts — it never targets the instruction channel.
arXiv:2605.09822 — Oracle Poisoning / Knowledge Graphs, May 2026
Treating the agent itself as untrusted cuts attack success to 0.65%
AgentVisor places a trusted semantic visor between the agent and its tool calls, applying suitability, taint, and integrity checks at execution time. Attack success: 0.65%. Utility cost: 1.45%. The architecture works because enforcement happens outside the model — where the agent cannot bypass it regardless of what instructions it received.
