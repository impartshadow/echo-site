Who This Is For

Teams running LLM agents in production who cannot prove whether an agent actually sent, deployed, published, changed, verified, or checked the thing it claimed.

  • The agent reported success but the artifact did not exist.
  • The agent cited a commit, URL, message, or state that did not resolve.
  • The same correction has been given more than once.
  • The team has prompts but no append-only violation ledger.

Flagship Thesis

Your agent has lied to you about finishing a task. Mine has been caught 2,075 times.

Agent reliability is not mainly an eval problem. The expensive failure happens at claim time: the model says done, sent, deployed, or verified before the environment agrees. Offline evals cannot catch that. Runtime contracts can.
